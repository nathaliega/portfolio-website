Countries API

So... how does it work?

My goal with this project was to create an API that allows the user to access some of the main economic indicators of a country.
To do so I laid out the structure I wanted the database to have, which was the following:
    Three models:
  1. Countries: includes all the economics indicators and has relationships with both of the other tables
  2. Continents: contains the continents names and ids
  3. Regions: contains the regions names and ids
This was done through the SQLAlchemy ORM, which allows us to communicate to the database using Python by acting like a “bridge” between them.
To write endpoints I used FastAPI, a framework typically used to build APIs. Each endpoint calls a CRUD (Create, Read, Update, Delete) function that allows the user to manipulate the database data in some way.

Finally, in regards to hosting, it is done using Docker to create a container which is an isolated lightweight Linux environment with all necessary dependencies installed.
